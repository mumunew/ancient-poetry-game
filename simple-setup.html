🛠️ Supabase 简易配置

避免 CORS 错误的手动配置方案

⚠️ 关于 CORS 错误

由于浏览器安全限制，无法直接从网页访问 Supabase Management API。我们提供以下手动配置方案，同样简单有效！

1 创建 Supabase 项目

1. 访问 Supabase 官网 并登录

2. 点击 "New Project" 创建新项目

3. 填写项目名称（如：poetry-game）

4. 选择数据库密码并记住它

5. 选择地区（推荐：Singapore 或 Tokyo）

6. 等待项目创建完成（约2-3分钟）

2 执行 SQL 脚本

1. 在项目控制台中，点击左侧菜单的 "SQL Editor"

2. 复制下面的 SQL 脚本并粘贴到编辑器中

3. 点击 "Run" 执行脚本

-- 创建游戏反馈表 CREATE TABLE IF NOT EXISTS game_feedback ( id BIGSERIAL PRIMARY KEY, player_name VARCHAR(100), contact_info VARCHAR(255), suggestions TEXT, score INTEGER DEFAULT 0 CHECK (score >= 0), accuracy DECIMAL(5,2) DEFAULT 0.00 CHECK (accuracy >= 0 AND accuracy <= 100), play_time INTEGER DEFAULT 0 CHECK (play_time >= 0), current_level INTEGER DEFAULT 1 CHECK (current_level >= 1), max_levels INTEGER DEFAULT 1 CHECK (max_levels >= 1), created_at TIMESTAMPTZ DEFAULT NOW(), user_agent TEXT, screen_resolution VARCHAR(50), -- 约束条件 CONSTRAINT valid_level_progress CHECK (current_level <= max_levels), CONSTRAINT valid_contact_or_suggestion CHECK ( contact_info IS NOT NULL OR suggestions IS NOT NULL ) ); -- 添加表注释 COMMENT ON TABLE game_feedback IS '古诗词游戏用户反馈数据表'; -- 启用行级安全 ALTER TABLE game_feedback ENABLE ROW LEVEL SECURITY; -- 允许匿名用户插入反馈 CREATE POLICY "Allow anonymous insert" ON game_feedback FOR INSERT TO anon WITH CHECK (true); -- 允许读取反馈 CREATE POLICY "Allow read feedback" ON game_feedback FOR SELECT TO anon USING (true); -- 创建性能索引 CREATE INDEX IF NOT EXISTS idx_game_feedback_created_at ON game_feedback(created_at DESC); CREATE INDEX IF NOT EXISTS idx_game_feedback_score ON game_feedback(score DESC); -- 创建统计视图 CREATE OR REPLACE VIEW feedback_stats AS SELECT COUNT(*) as total_feedback, AVG(score) as avg_score, MAX(score) as max_score, AVG(accuracy) as avg_accuracy, AVG(play_time) as avg_play_time FROM game_feedback;
3 获取 API 密钥

1. 在项目控制台中，点击左侧菜单的 "Settings"

2. 选择 "API" 选项卡

3. 复制以下信息：

  • Project URL：项目的 API 地址
  • anon public：匿名访问密钥

🧪 测试配置

6 测试配置

1. 打开游戏页面：index.html

2. 玩几关游戏

3. 点击"提交反馈"按钮

4. 填写反馈信息并提交

5. 在 Supabase 控制台的 "Table Editor" 中查看 game_feedback 表，确认数据已成功插入

🔍 验证方法

在浏览器开发者工具的控制台中，应该能看到 "✅ 反馈提交成功" 的消息，而不是错误信息。

💡 常见问题

  • 403 错误：检查 RLS 策略是否正确设置
  • 连接失败：确认 Project URL 和 Anon Key 是否正确
  • 数据未插入：检查表结构是否正确创建
